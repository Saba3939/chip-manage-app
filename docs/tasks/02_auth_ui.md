# Task 02: 認証UI

## 目的
ログイン・登録画面のUIを構築し、モックデータで画面遷移を確認する。

## 実装内容

### 1. ログイン画面（`app/(auth)/login/page.tsx`）
**表示項目**
- アプリタイトル・ロゴ
- メールアドレス入力フィールド
- パスワード入力フィールド
- ログインボタン
- 「アカウントを作成」リンク（登録画面へ）
- Googleログインボタン（後で実装）

**デザイン方針**
- カード形式の中央配置レイアウト
- モバイルファーストデザイン
- shadcn/ui の Card, Input, Button を使用

### 2. 登録画面（`app/(auth)/register/page.tsx`）
**表示項目**
- アプリタイトル
- 表示名入力フィールド
- メールアドレス入力フィールド
- パスワード入力フィールド
- パスワード確認入力フィールド
- 登録ボタン
- 「ログインはこちら」リンク

**バリデーション（UI側のみ）**
- 必須項目チェック
- メール形式チェック
- パスワード長さチェック（8文字以上）
- パスワード一致チェック

### 3. 認証レイアウト（`app/(auth)/layout.tsx`）
**特徴**
- シンプルな背景
- 中央配置
- ヘッダー・フッターなし

### 4. モック動作
- ログインボタンクリック → ダッシュボードへ遷移
- 登録ボタンクリック → ダッシュボードへ遷移
- 実際の認証処理はTask 10で実装

## 使用コンポーネント
- Card, CardHeader, CardTitle, CardContent
- Input
- Label
- Button
- Form（react-hook-form + zod）

## 完了条件
- [ ] ログイン画面が表示される
- [ ] 登録画面が表示される
- [ ] 画面間の遷移が動作する
- [ ] フォームバリデーションが機能する（エラー表示）
- [ ] モバイル・デスクトップで適切に表示される

## 動作確認
- `/login` にアクセスして画面が表示される
- `/register` にアクセスして画面が表示される
- フォーム入力時にバリデーションエラーが表示される
- ボタンクリックでダッシュボードに遷移する（モック）
