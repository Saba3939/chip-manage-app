# Task 08: 清算画面UI

## 目的
セッション終了時の清算画面UIを構築し、増減と清算額を表示する。

## 実装内容

### 1. 清算画面（`app/(main)/session/[id]/settlement/page.tsx`）

**表示項目**
- ページタイトル: 「清算」
- セッション情報
  - セッション名
  - 開催日時
  - 参加人数
- 清算テーブル
  - カラム:
    - 参加者名
    - 初期チップ
    - 最終チップ
    - 増減（+/- 表示、色分け）
    - 清算額（円）
- サマリー
  - チップ総数（検証用）
  - 清算額合計（0円になるはず）
- アクションボタン
  - 「ダッシュボードに戻る」ボタン
  - 「履歴を見る」ボタン

### 2. コンポーネント設計

**`components/settlement/SettlementTable.tsx`**
- 清算データのテーブル表示
- ソート機能（増減額でソート）
- レスポンシブ対応（モバイルではカード形式）

**`components/settlement/ParticipantRow.tsx`**
- 個別の参加者行
- 増減額に応じた色分け
  - プラス: 緑色
  - マイナス: 赤色
  - ゼロ: グレー

**`components/settlement/SettlementSummary.tsx`**
- サマリー情報表示
- チップ総数の検証結果
  - 初期チップ × 参加者数 = 最終チップ合計
  - 不一致がある場合は警告表示

### 3. レート設定ダイアログ（オプション）
- セッション作成時にレートを設定していない場合
- 清算画面で初めてレート入力
- デフォルト: 10円

### 4. モックデータ
- 各参加者の初期チップ・最終チップ
- レート設定値
- 清算額の計算結果

### 5. 計算ロジック（フロントエンド）
```
増減 = 最終チップ - 初期チップ
清算額 = 増減 × レート
```

## 使用コンポーネント
- Card
- Table, TableHeader, TableBody, TableRow, TableCell
- Badge
- Button
- Alert（検証エラー表示用）

## 完了条件

- [ ] 清算テーブルが表示される（モック）
- [ ] 増減額が色分けされている
- [ ] 清算額が計算されている
- [ ] サマリーが表示される
- [ ] モバイル表示が適切に動作する

- 清算画面へ遷移する
- 清算テーブルが表示される
- 増減額と清算額が正しく表示される
- チップ総数の検証が表示される
