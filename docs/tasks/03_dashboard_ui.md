# Task 03: ダッシュボードUI

## 目的
ログイン後の初期画面となるダッシュボードのUIを構築する。

## 実装内容

### 1. ダッシュボード画面（`app/(main)/dashboard/page.tsx`）

**表示項目**
- ユーザー情報表示
  - 表示名
  - ログアウトボタン
- アクションカード
  - 「新しいセッションを作成」ボタン
  - 「セッションに参加」ボタン
- アクティブセッション一覧（モックデータ）
  - セッション名
  - 参加人数
  - ステータス（待機中/進行中）
  - クリックでセッション詳細へ遷移
- 最近の履歴（モックデータ、3件程度）
  - 日時
  - セッション名
  - 結果（+/-チップ数）

**デザイン方針**
- グリッドレイアウト（モバイルは1カラム、デスクトップは2カラム）
- カード形式で情報を整理
- アクションボタンは目立つデザイン

### 2. メインレイアウト（`app/(main)/layout.tsx`）

**共通要素**
- ヘッダー
  - アプリロゴ
  - ユーザー名表示
  - メニュー（ハンバーガーメニュー）
- ナビゲーション
  - ダッシュボード
  - 履歴
  - 設定（将来的に）

### 3. モックデータの定義
`lib/mock-data.ts` に以下を定義：
- サンプルユーザー
- サンプルセッション（3-5件）
- サンプル履歴データ

## 使用コンポーネント
- Card
- Button
- Badge（ステータス表示用）
- Avatar（ユーザーアイコン）
- Separator

## 完了条件
- [ ] ダッシュボード画面が表示される
- [ ] ヘッダー・ナビゲーションが機能する
- [ ] アクティブセッション一覧が表示される（モック）
- [ ] 履歴が表示される（モック）
- [ ] 各ボタンからの遷移が動作する

## 動作確認
- `/dashboard` にアクセスして画面が表示される
- 「新しいセッションを作成」クリック → セッション作成画面へ遷移
- 「セッションに参加」クリック → 参加画面へ遷移
- セッションカードクリック → セッション詳細画面へ遷移
- レスポンシブ表示が適切に動作する
