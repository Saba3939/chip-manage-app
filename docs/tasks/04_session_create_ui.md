# Task 04: セッション作成UI

## 目的
ホストがゲームセッションを作成する画面のUIを構築する。

## 実装内容

### 1. セッション作成画面（`app/(main)/session/create/page.tsx`）

**表示項目**
- ページタイトル「新しいセッションを作成」
- フォーム
  - セッション名（任意）
    - プレースホルダー: 「今日のポーカー」など
  - 初期チップ数
    - デフォルト: 1000
    - 選択肢: 500, 1000, 2000, 5000
  - 最大参加人数
    - デフォルト: 10
    - 選択肢: 2-20人
  - レート設定（任意）
    - 1チップあたりの円換算
    - プレースホルダー: 10円
- 作成ボタン
- キャンセルボタン

**バリデーション**
- 初期チップ数: 正の整数
- 最大参加人数: 2以上
- レート: 正の数値（任意）

**モック動作**
- 作成ボタンクリック → セッション詳細画面へ遷移
- モックのセッションIDを生成

### 2. コンポーネント設計

**`components/session/SessionCreateForm.tsx`**
- フォームロジックを管理
- react-hook-form + zod でバリデーション

**`components/session/ChipAmountSelector.tsx`**
- チップ数選択用のカスタムコンポーネント
- ボタンで選択 or 手動入力

## 使用コンポーネント
- Card
- Form, Input, Label
- Select（参加人数選択）
- Button
- RadioGroup（チップ数選択）

## 完了条件
- [ ] セッション作成画面が表示される
- [ ] フォームが入力できる
- [ ] バリデーションが動作する
- [ ] 作成ボタンでセッション詳細画面へ遷移する
- [ ] キャンセルボタンでダッシュボードに戻る

## 動作確認
- `/session/create` にアクセスして画面が表示される
- 各フィールドに入力できる
- バリデーションエラーが表示される
- 作成ボタンでセッション詳細画面へ遷移（モックID使用）
