# Task 05: セッション参加UI

## 目的
QRコードスキャンまたはID入力でセッションに参加する画面のUIを構築する。

## 実装内容

### 1. セッション参加画面（`app/(main)/session/join/[id]/page.tsx`）

**2つのモード**

#### モード1: QRスキャン
- カメラプレビュー表示
- QRコード読み取り中の状態表示
- 読み取り成功 → セッション詳細画面へ遷移

#### モード2: ID手動入力
- セッションID入力フィールド
- 参加ボタン
- セッションが見つからない場合のエラー表示

**タブ切り替え**
- 「QRコードスキャン」タブ
- 「IDを入力」タブ

### 2. コンポーネント設計

**`components/qr/QRScanner.tsx`**
- `html5-qrcode` を使用
- カメラアクセス許可の処理
- QRコード検出時のコールバック
- エラーハンドリング（カメラアクセス拒否など）

**`components/session/JoinByIdForm.tsx`**
- ID入力フォーム
- バリデーション

### 3. パーミッション処理
- カメラアクセス許可のリクエスト
- 拒否された場合の説明表示
- 手動入力への誘導

### 4. モック動作
- QRコード読み取り → URLからセッションIDを抽出
- モックのセッションIDで詳細画面へ遷移
- 存在しないIDの場合はエラー表示

## 使用コンポーネント
- Card
- Tabs, TabsList, TabsTrigger, TabsContent
- Input
- Button
- Alert（エラー表示用）

## 外部ライブラリ
- `html5-qrcode`: QRコード読み取り

## 完了条件
- [ ] セッション参加画面が表示される
- [ ] QRスキャンタブが動作する（カメラ起動）
- [ ] ID入力タブでフォームが動作する
- [ ] モックデータでセッション詳細へ遷移できる
- [ ] カメラアクセス拒否時のエラー処理が動作する

## 動作確認
- `/session/join/[id]` にアクセスして画面が表示される
- QRスキャンタブでカメラが起動する
- ID入力タブでIDを入力できる
- 参加ボタンでセッション詳細へ遷移する

## トラブルシューティング
- **カメラが起動しない**: HTTPSまたはlocalhostで動作確認
- **QRコード読み取りが遅い**: `html5-qrcode`の設定を調整
